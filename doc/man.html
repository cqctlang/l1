Types

Cinquecento distinguishes the name of a C type from its definition. Type names are syntactic constructions that appear in C declarations, such as int*. They determine the C operations, such as pointer dereference, that may be applied to a value of the named type. Evaluation of these operations, however, requires knowledge of the layout and encoding of the value in memory. Type definitions provide this information.

For example, the type name int* identifies the type pointer-to-int. A value of this type may be dereferenced to yield a value of type int, but evaluation of the dereference operation depends on knowing the size of the pointer and the size and encoding of int. This information comes from the definition of int*. There are many possible definitions. One definition for int* may be a 64-bit address of a 64-bit integer encoded in big-endian order, while another may be a 32-bit address of a 32-bit integer encoded in little-endian order. One component of a Cinquecento name space is a function that maps type names to type definitions.

Type definitions are encapsulated in first-class Cinquecento values called ctypes. Ctypes for for arithmetic types define the size (number of bytes) and encoding (byte order) of the values represented by the type. Ctypes for aggregate types (structures and unions) include the overall size of the aggregate and a specification of each field that includes the name of the field, its offset from the beginning of the aggregate, and its type definition.
